# Cat-A-Log Demo Video — Filming Guide

A structured script for manually recording a ~3 minute walkthrough/demo video. Designed for mobile screen recording (portrait, 9:16) with optional voiceover added in post.

---

## Pre-Recording Setup

### Environment prep

- [ ] Seed the database with 5–7 cats (varied names, photos, locations, welfare flags)
- [ ] Have a second account logged in on another device or incognito (for "community" feel)
- [ ] Use a clean phone/emulator with no notifications visible
- [ ] Turn on Do Not Disturb
- [ ] Dark mode OFF (orange accent pops better on light)
- [ ] GPS/location services enabled (or mock location set to a neighborhood with seeded data)
- [ ] Clear browser cache so first-load animations fire

### Recording setup

- Portrait orientation (1080×1920)
- Screen recording at 60fps if possible (smoother scrolling)
- Hide the status bar or keep it minimal
- If using a phone, consider a screen recorder like AZ Recorder (Android) or built-in iOS

## ACT 3 — First Tag Flow (0:25–1:25) ⭐ Hero sequence

**Type:** SCREEN RECORDING — the core "tag a cat" experience  
**This is the money shot. Take your time, be deliberate.**

### What to film (in order):

1. **Start on the map page** — show the bottom nav, a few existing cat markers bouncing in
2. **Tap the center FAB (Tag button)** — the elevated orange button
3. **Photo screen** — tap "Take Photo" or select from gallery. Pick a clear cat photo.
4. **Wait for the upload** — show the loading state briefly
5. **Candidates screen** — nearby cats are shown first (by proximity). Show 2-3 candidate cards.
   - Tap "None of these" (we're tagging a NEW cat for drama)
6. **Name screen** — type a fun name (e.g., "Mochi", "Captain Whiskers"). Show the random name generator button for flair.
7. **Details screen** — toggle ear-tipped if relevant, add a short note, confirm GPS pin on the mini-map
8. **Submit** — watch the success toast + the catch card reveal animation

### Filming tips:

- Pause 1–2 seconds on each screen before interacting (gives viewer time to read)
- Thumb should enter from the bottom of frame (natural mobile grip)
- If the cat photo is a real cat from your neighborhood, even better for authenticity
- Show the step dots progressing at the top

### Caption suggestions (for text overlay in editing):

| ----------------- | ------------------------- |
| When camera opens | "Snap a photo"            |
| Candidates appear | "Nearby cats shown first" |
| Name screen       | "Give them a name"        |
| On submit         | "Tagged! 🎉"              |

## ACT 4 — Map Comes Alive (1:25–1:45)

**Type:** SCREEN RECORDING

### What to film:

1. **Return to map** (should happen automatically after tagging)
2. **Show the newly tagged cat marker appearing** — bounce-in animation
3. **Pan/zoom the map** to show other cats in the area (the seeded data)
4. **Tap a cat marker** — show the preview card sliding up from the bottom
5. **Show clustering** — zoom out so markers cluster together, zoom in to uncluster
6. **Tap "Search this area"** pill if visible

### Filming tips:

- Smooth, slow pan gestures — don't jerk the map around
- Let markers settle before interacting
- Show the glassmorphic preview card in full glory

### Caption suggestions:

| Moment           | Caption              |
| ---------------- | -------------------- |
| Map with markers | "Every cat, mapped"  |
| Tap a marker     | "Tap to see details" |

## ACT 5 — AI Match + Welfare Flag (1:45–2:25) ⭐ Technical differentiator

**Type:** SCREEN RECORDING — one continuous take

### What to film:

1. **Start a NEW tag flow** with a photo of a cat that looks similar to one already in the database
2. **Show the candidates screen** — nearby cats appear by proximity first
3. **Tap the "Check with AI" button** — AI re-ranks candidates by visual similarity, highlighting the best match
4. **Tap "This is the same cat!"** — confirm the match
5. **Show the match-found screen** — sighting count goes up (e.g., "2 → 3 sightings")
6. **Expand the community edit section** (chevron/accordion below the card)
7. **Tap a health flag pill** — e.g., "Needs medical" — show the amber highlight appearing
8. **Save** — the flag is now on the cat's record
9. **Return to map** — marker shows the welfare indicator

### Filming tips:

- Use two photos of the same cat (or visually similar cats) to make the AI match feel magical
- The key moment is tapping "Check with AI" and watching the ranking change — give it breathing room
- Let the similarity percentage/ranking be visible after AI runs
- Keep rolling after the match — the transition into community editing should feel natural ("oh, this cat looks rough")
- The amber/red color coding on health flags should be clearly visible
- Total ~40 seconds: don't linger, but don't rush the two "wow" beats (AI re-rank + flag save)

### Caption suggestions:

| Moment                   | Caption                         |
| ------------------------ | ------------------------------- |
| Nearby cats appear       | "Nearby cats shown first"       |
| Tap Check with AI        | "AI ranks by visual similarity" |
| Confirm tap              | "One tap to confirm"            |
| After confirmation       | "Sighting #3 logged"            |
| Expanding community edit | "Spot something wrong?"         |
| Tapping health flag      | "Flag it for the community"     |
| After saving             | "Now everyone nearby knows"     |

## ACT 6 — Profile & Collection (2:25–2:40)

**Type:** SCREEN RECORDING

### What to film:

1. **Navigate to Profile** (bottom nav, right icon)
2. **Show the profile page** — avatar, username, bio, tags count
3. **Featured cat card** — the hero photo with tier badge (if one is set as featured)
4. **Scroll to "My Cats" list** — show the rows with photos, names, sighting counts, welfare tag pills
5. **Star a cat** — show setting a cat as "Featured" (the star button)
6. **Tap share** — show the share profile button generating a shareable card

### Where's the catch card?

The full trading-card-style catch card (with motifs, tier design) appears as a reward **right after tagging** — it was already shown at the end of Acts 3 and 5. The profile shows your collection as a list, not as cards.

### Filming tips:

- The featured cat card with its tier badge (Common/Rare/Legendary) and glow is the visual highlight here
- Show a cat with multiple sightings so the tier is visibly upgraded
- Brief section — this is a wind-down before the closing

## Post-Production Notes

### Editing rhythm

- **Footage clips:** Cut on action (finger tap = cut point)
- **Text cards:** Hold 4-5 seconds each line
- **Transitions:** Simple cross-dissolve or quick cut. Avoid flashy wipes.
- **Orange wipe:** If you want to keep the remotion aesthetic, use a solid orange (#f97316) slide-in between acts

### Audio

- Background music: lo-fi, warm acoustic, or light electronic (royalty-free)
- Optional voiceover: conversational tone, not corporate. Match the "warm, playful, communal" brand.
- Sound effects: subtle tap sounds on UI interactions (optional, adds polish)

### Text overlays

- Font: Varela Round for headlines, Inter/Geist for body
- Position: bottom-third of screen (avoid obscuring the UI)
- Background: semi-transparent dark pill (like the app's glassmorphic treatment)

### Tech callouts (optional lower-third badges)

If this is for a technical audience (hackathon judges, portfolio):

- "CLIP Embeddings — Photo → Vector → Match"
- "Next.js 16 + Supabase + pgvector"
- "PWA — works offline"

Place these as brief 3-second lower-thirds during relevant footage.

## Recording Order (efficiency)

Film in this order to minimize re-navigation:

1. **Map overview** (Act 4) — before you tag anything new
2. **First tag flow** (Act 3) — fresh tag, tap "None of these"
3. **AI match + Welfare flag** (Act 5) — second tag with similar photo, confirm match, expand community edit, add health flag (one continuous take)
4. **Profile + cards** (Act 6) — natural endpoint
5. **PWA install** (bonus) — if doing it

Then assemble in story order during editing.
